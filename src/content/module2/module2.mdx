# Módulo 2: Composition API y SFC

En este módulo profundizaremos en la **Composition API** de Vue 3 y las mejoras en los **Single File Components (SFC)**. Aprenderás a organizar tu código de manera más eficiente y a crear aplicaciones más mantenibles.

## 🎯 Objetivos del módulo

Al completar este módulo dominarás:

- **Composition API avanzada**: Patrones y mejores prácticas
- **Composables personalizados**: Creación y reutilización de lógica
- **Gestión de estado local**: Con `ref()`, `reactive()` y `computed()`
- **Ciclo de vida en Composition API**: Hooks y su uso efectivo
- **SFC mejorados**: Nuevas características y optimizaciones

## 📚 Contenido del módulo

### Sección 1: Composition API Avanzada

Exploraremos conceptos avanzados como:

- **Composables**: Funciones reutilizables para lógica compartida
- **Provide/Inject**: Inyección de dependencias en Composition API
- **Watchers avanzados**: `watch`, `watchEffect` y sus casos de uso
- **Refs avanzadas**: `toRefs`, `unref`, `isRef` y utilidades

## 🧩 Composables: El poder de la reutilización

Los composables son el corazón de la Composition API. Te enseñaremos a:

- **Extraer lógica común** en funciones reutilizables
- **Crear abstracciones útiles** para tu aplicación
- **Combinar múltiples composables** efectivamente
- **Testear composables** de forma aislada

## 🔄 Patrones de diseño

Aprenderás patrones comunes como:

- **useCounter**: Gestión de contadores
- **useFetch**: Llamadas a APIs
- **useLocalStorage**: Persistencia local
- **useMouse**: Tracking de eventos del mouse
- **useToggle**: Estados booleanos

## 🎨 SFC Mejorados

Vue 3 introduce mejoras significativas en los SFC:

- **`<script setup>`**: Sintaxis más concisa
- **CSS v-bind**: Variables CSS reactivas
- **Scoped slots mejorados**: Mayor flexibilidad
- **Mejor soporte para TypeScript**

## 🚀 Ventajas de la Composition API

- **Mejor organización**: Lógica agrupada por funcionalidad
- **Reutilización**: Composables compartibles entre componentes
- **TypeScript**: Inferencia de tipos mejorada
- **Tree-shaking**: Mejor optimización del bundle
- **Testabilidad**: Lógica más fácil de testear
