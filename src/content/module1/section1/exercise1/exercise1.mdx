# Ejercicio 1: De Options API a Composition API

## üìù Tarea

Reescribe el siguiente componente contador de Vue 2 (Options API) a Vue 3 usando Composition API con `<script setup>`:

```vue
<template>
  <div>
    <h2>{{ title }}</h2>
    <p>Valor: {{ counter }}</p>
    <button @click="increment">Incrementar</button>
    <button @click="reset">Reset</button>
    <p v-if="isEven">El n√∫mero es par</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      title: 'Contador Vue 2',
      counter: 0
    }
  },
  methods: {
    increment() {
      this.counter++
    },
    reset() {
      this.counter = 0
    }
  },
  computed: {
    isEven() {
      return this.counter % 2 === 0
    }
  }
}
</script>
```

### üí° Pistas

- Usa `ref()` para datos reactivos
- Usa `computed()` para propiedades calculadas
- Las funciones se declaran directamente en el script
- No olvides el `.value` en las refs

## ‚úÖ Soluci√≥n

```vue
<template>
  <div>
    <h2>{{ title }}</h2>
    <p>Valor: {{ counter }}</p>
    <button @click="increment">Incrementar</button>
    <button @click="reset">Reset</button>
    <p v-if="isEven">El n√∫mero es par</p>
  </div>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue'

const title = ref('Contador Vue 3')
const counter = ref(0)

const increment = () => {
  counter.value++
}

const reset = () => {
  counter.value = 0
}

const isEven = computed(() => counter.value % 2 === 0)
</script>
```

### üîç Explicaci√≥n de la soluci√≥n

1. **`ref()`** para datos reactivos: Reemplaza `data()` del Options API
2. **`computed()`** para propiedades calculadas: Mantiene la misma funcionalidad pero con nueva sintaxis
3. **Funciones directas**: Los m√©todos ahora son funciones constantes declaradas directamente
4. **`.value`** para acceder a refs: Necesario para leer/escribir valores de referencias reactivas
5. **Imports expl√≠citos**: Vue 3 requiere importar las funciones que uses desde 'vue'
